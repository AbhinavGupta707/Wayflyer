# Keeper

**An agentic returns-rescue system for the streetwear brand Pretty Fly.**
Built for the Wayflyer × Fin hackathon ("Build the Future of eCommerce").

Returns are every store's worst nightmare. For clothing, the number one reason an item comes back is the wrong size. Over two years, returns cost Pretty Fly **£602,000**, and more than half of that, **£306,000**, was nothing but sizing.

Keeper intercepts the return at the moment of intent, decides the highest-margin save (an **exchange** beats a **refund**), executes it across chat and voice, and learns from every single return so the next decision is smarter. Every number and every decision is deterministic; the AI only ever handles the words.

---

## The proof (LLM-free backtest, reproducible to the penny)

Run against Pretty Fly's entire 24-month history, all 5,843 real refunds, with point-in-time stock:

| | Amount | What it means |
|---|---|---|
| Size-driven refunds | **£305,692** | 2,393 of 5,843 refunds were just the wrong size |
| Recoverable today | **£182,951** | the corrective size was in stock the moment they returned |
| Margin retained | **£108,001** | pure margin Keeper keeps by swapping instead of refunding |
| Lost to stockout | **£67,167** | a fix existed but was out of stock — an inventory signal, not a returns one |

Headline insight: **trainers run small** and dominate the recoverable pile.

Regenerate any time:
```bash
cd keeper/apps/api && python -m keeper_api.engine.backtest
```

## How it works

A return triggers a swarm of small, specialised agents (orchestrated over open-source models on **Fireworks**, so no customer data ever leaves the company to a third party like OpenAI or Anthropic):

```
intake → triage → context → fit → inventory → economics → governor → concierge → memory → decision
```

- **Triage** classifies why the item is coming back and reads sentiment.
- **Context** loads the customer's *passport* (a memory profile of all 22,440 customers: sizes kept, sizes returned, discount sensitivity) plus the SKU genome and live stock.
- **Fit / Inventory / Economics / Governor** pick the corrective size, confirm it is in stock, weigh refund vs exchange, and check policy.
- **Concierge** writes the offer; the customer can accept by click or by talking to an **ElevenLabs** voice concierge that books the exchange hands-free.
- **Memory** writes the outcome back: every return updates the customer profile and flags SKU patterns for the buying team.

**The hard rule:** a deterministic engine owns every monetary calculation and the decision. The LLM layer only generates language and drives actions. So the money is never hallucinated and the customer is never put at risk.

## Tech stack

- **Backend:** Python 3.11+, FastAPI + Uvicorn. Decision engine and backtest use only the standard library (no pandas needed at runtime), so the proof numbers are fully reproducible.
- **Agents:** LangGraph swarm over Fireworks OSS models (gated off by default; deterministic templated stream is the reliable demo path).
- **Voice:** `@elevenlabs/client` (WebRTC) browser concierge.
- **Frontend:** Next.js (App Router) + React + TypeScript + Tailwind + Framer Motion + React Flow + Recharts.

## Quickstart

**Prerequisites:** Python 3.11+ and Node 18+.

### 1. Backend (API + the proof)

```bash
cd keeper/apps/api
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# build the feature stores + ledger (stdlib only, ~10s)
python -m keeper_api.etl.build          # -> cache/{passports,genomes,...}.json
python -m keeper_api.engine.backtest    # -> cache/ledger.json

# create your .env (see "Environment variables" below), then run:
uvicorn keeper_api.main:app --reload --port 8000
```

API is on `http://localhost:8000`. Key routes:
`GET /api/health · /api/ledger · /api/customers · /api/passports/galaxy · /api/rescue/{id}/steps`,
`POST /api/returns/intake · /api/rescue/{id}/respond`, `WS /api/rescue/{id}/stream`.

### 2. Frontend

```bash
cd keeper/apps/web
npm install
npm run dev                              # http://localhost:3000
```

> The whole demo runs **without any API keys**: the agents fall back to a deterministic templated stream and the voice modal shows a text transcript with Yes/No. Keys only unlock the live LLM agents and the spoken voice concierge.

## Environment variables

Create a file at **`keeper/apps/api/.env`** (gitignored — never committed). All values are optional:

```dotenv
# ElevenLabs voice concierge (browser widget)
ELEVENLABS_API_KEY=sk_...
ELEVENLABS_AGENT_ID=agent_...

# Fireworks — only needed if you flip the live LLM agents on
FIREWORKS_API_KEY=fw_...
```

| Variable | Location | Purpose | Default |
|---|---|---|---|
| `ELEVENLABS_API_KEY` | `keeper/apps/api/.env` | voice concierge | unset (text fallback) |
| `ELEVENLABS_AGENT_ID` | `keeper/apps/api/.env` | the Conversational-AI agent | unset |
| `FIREWORKS_API_KEY` | `keeper/apps/api/.env` | live agent swarm | unset (templated) |
| `KEEPER_LIVE_LLM` | shell env | set to `1` to enable live Fireworks agents | off |
| `NEXT_PUBLIC_API_BASE` | `keeper/apps/web/.env.local` | API base URL for the web app | `http://localhost:8000` |

A fuller template (including optional Twilio for an on-stage phone call) lives at
`keeper/apps/api/keeper_api/voice/.env.example`.
